# Portfolio & Resume Project Context

This repo contains two Maverick epics:

- `portfolio`: public portfolio website and project presentation.
- `resume`: official resume and cover-letter materials for job applications.

## Shared Career Context

The shared goal is career proof for software engineering opportunities. Portfolio and Resume may share career facts, project names, experience details, and accomplishments, but they optimize for different outputs.

## Portfolio Epic Role

The portfolio website should stay clean, current, simple, and authentic. All projects should look maintained and intentional. Current and classic projects may appear higher in the page hierarchy, but older projects should not look abandoned.

Public naming rule: use **Astra** for the public-facing network intelligence project. Use **Netwise** only for internal repo/config references.

## Resume Epic Role

The resume epic is not a website coding lane. It maintains official resumes, cover letters, experience inventory, formatting goals, and job-application tailoring rules.
